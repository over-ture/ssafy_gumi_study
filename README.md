* **주 3회 이상 필수 제출**: 바쁜 일정이 있더라도 최소 주 3회는 백준 문제를 풀고 업로드합니다.
* **난이도별 운영**: 본인의 수준에 맞춰 트랙을 선택하여 진행합니다.

```text

  │   └── 📁 260126_문제번호

  └── 📁 Bronze

```

## 🚀 상세 참여 방법 (Step-by-Step)

우리 스터디는 **Fork & Pull Request** 방식을 사용하며, 매주 **토요일**에 일괄 머지를 진행합니다.

### 1. 초기 설정 (최초 1회)

1. 상단의 **`Fork`** 버튼을 눌러 본인의 계정으로 레포지토리를 복제합니다.
2. 복제된 본인의 레포지토리를 로컬 PC에 클론합니다.
    ```bash
    git clone https://github.com/본인계정/ssafy_gumi_study.git
    ```


3. 원본 레포지토리를 `upstream`이라는 이름으로 등록합니다 (최신 상태 유지를 위함).
    ```bash
    git remote add upstream https://github.com/원본계정/ssafy_gumi_study.git
    ```



### 2. 문제 풀이 및 업로드

1. **최신 상태 동기화**: 작업 전 원본 레포지토리의 최신 코드를 가져옵니다.
    ```bash
    git checkout master
    git pull upstream master
    ```


2. **브랜치 생성**: 본인의 ID와 날짜를 조합해 브랜치를 만듭니다. (예: `feat/memberID-week1`)
    ```bash
    git checkout -b feat/memberID-week1
    ```


3. **코드 작성**: 지정된 폴더 구조에 맞춰 문제를 풀고 저장합니다.
* 구조: `본인ID / [Bronze|Silver] / 날짜_문제번호 / 문제번호.py`


4. **커밋 및 푸시**: 컨벤션에 맞춰 커밋 후 본인의 레포지토리에 올립니다.
    ```bash
    git add .
    git commit -m "[Silver 4] 11399_ATM / 20분 / 그리디 기초"
    git push origin feat/memberID-week1
    ```



### 3. Pull Request 생성 (주중 상시)

1. 본인의 GitHub 레포지토리에서 **`Compare & pull request`** 버튼을 누릅니다.
2. **Title**: `[본인ID] 1월 4주차 문제 풀이 제출`
3. **Content**: 이번 주에 푼 문제 리스트나 어려웠던 점을 간단히 적습니다.
4. `Create pull request`를 눌러 제출합니다.

### 4. 코드 리뷰 및 토요일 머지 (매주 토요일)

* **주중**: 다른 멤버의 PR에 들어가 코드를 구경하고, 궁금한 점은 댓글(Review)을 남깁니다.
* **토요일**: 스터디장(또는 정해진 담당자)이 제출된 PR의 컨벤션 등을 확인한 후, 이상이 없으면 `master` 브랜치로 **일괄 머지**를 진행합니다.
* 머지가 완료되면 본인의 로컬 `master` 브랜치를 다시 최신화합니다.
