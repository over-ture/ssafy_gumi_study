## 📌 문제 개요
 문제는 여러 개의 전깃줄이 얽혀 있을 때, 서로 교차하지 않게 하기 위해 제거해야 하는 전깃줄의 최소 개수를 구하는 문제입니다.
모든 전깃줄이 서로 교차하지 않도록 만들기 위해 제거해야 하는 최소 전깃줄 수를 구합니다.


---

## 💡 해결 프로세스


 1. 제거할 최소 개수 = 전체 전깃줄 수 - 교차하지 않고 남길 수 있는 최대 개수.
 2. DP를 사용하여 풉니다. 상태정의는 dp[now Left Story][now Right Story]일때,저장하느 값은 '교차하지 않고' 연결 가능한 최대한의 전선 개수 입니다.
 3. 현재 전기줄의 위치에 도달 했을때  이전 층까지의 최대 연결 값에 1을 더합니다. 
 4. 이전 층까지 교차하지 않은 최대 연결 전선 수가 저장되어 있고, 왼쪽 오른쪽 두 쌍 모두 이전 층의 정보를 바탕으로 연산하므로 현재 전선 위치 쌍과  이전에 고려된 모든 전선들은 절대 교차하지 않습니다.
---

## 💻 코드 구조 상세 (Core Logic)

🔍 직전 층까지의 연결 가능한 최대 전선 수에 자기 자신 1 을 더합니다.('한 쌍'에 대해 증가하는 부분수열 적용) 
```Java
         // dp[left story][ right story] -> (가능한) 최대 전기줄 연결 개수
		for (int i = 1; i <= 500; ++i) {
			for (int j = 1; j <= 500; ++j) {
				dp[i][j]  =Math.max(dp[i - 1][j-1],  Math.max(dp[i - 1][j], dp[i][j - 1]) );
				int ret = dp[i][j] ;
				for (int k = 0; k < n; ++k) {
					if (starts[k] != i)
						continue;
					if (ends[k] != j)
						continue;
					dp[i][j] = ret+1;
				
				}

			}
		}
		System.out.println(n - dp[500][500]);
```

⚠️ 주의 및 회고
 저는 왼쪽 높이와 오른쪽 높이의 상태를 모두 고려한 풀이를 생각하였지만(N^3), 상태를 정의할 떄 왼쪽 기반으로 오름차순 정렬해놓고 K 번째 전기줄 이전까지 최대한 몇 개의 전선을 연결할 수 있는지 조사함으로써 N^2 풀이가 가능했습니다. 문제는 LIS라는 알고리즘으로 치환이 가능해 크기가 커지면 이분탐색으로 풀 수 있습니다.
